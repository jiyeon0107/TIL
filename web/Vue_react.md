## 배운 것
Vue의 반응형 시스템에서 JS 값 변경이 화면에 반영되는 흐름(정방향)과, 사용자 입력이 JS로 전달되는 흐름(역방향)이 서로 다르게 동작한다는 것과, v-model이 이 둘을 자동으로 연결해준다는 것.

## 왜 중요한가?
정방향/역방향 흐름을 구분해서 이해해야 v-model 없이 직접 바인딩을 구현하거나, "값은 바뀌었는데 화면이 안 바뀐다" 같은 반응형 관련 버그를 정확히 디버깅할 수 있다.

## 새롭게 알게 된 정보
1. 정방향: JS(Vue)의 값이 변경되면 DOM에다가 넣고 렌더링 엔진이 DOM에 있는 정보 가져다가 써서 그린다.
2. 역방향: 사용자가 타이핑하면 브라우저가 입력창 화면에 바로 반영하고 이벤트 발생 시켜서 JS가 event로 값을 읽어온다.
3. Vue의 v-model을 사용하면 정방향/역방향이 저절로 적용된다.

## 예시

**Vue의 반응형 시스템**
```
[정방향]
① 값 변경          : ch_id.value = 2
② JS(Vue)가 알아챔  : set 트랩 발동 → 렌더 effect 재실행
③ DOM에 넣음        : Vue가 직접 DOM에 씀 (el.textContent = "2")
④ 렌더링 엔진이 그림 : DOM의 바뀐 정보를 가져다 Layout→Paint→Composite로 화면에 반영

한 줄로 요약하면: "값 변경 → Vue가 알아챔 → Vue가 DOM에 반영 → 렌더링 엔진이 DOM 보고 화면 그림."

[역방향]
① 키보드 입력
② 브라우저 엔진이 <input> 요소의 값을 "자체적으로" 바로 바꿈
   (이 시점에 이미 화면엔 글자가 보임 — JS는 아직 아무것도 안 함!)
③ 브라우저가 "input"이라는 DOM 이벤트를 발생시킴
④ JS가 그 요소에 미리 등록해둔 이벤트 리스너가 실행됨
   → e.target.value를 읽어서 ch_id.value = e.target.value 로 대입
⑤ 이제서야 Vue reactivity의 set 트랩이 발동 (지금까지 얘기한 정방향 흐름 시작)
```
