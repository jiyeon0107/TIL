## 배운 것
Java의 역사

## 왜 중요한가?
Java의 실행환경 및 역사를 이해해야 언어를 익히는 데 도움이 된다.

## 새롭게 알게 된 내용
- 자바는 여러 플랫폼에서 수행할 수 있는 개발 언어를 목표로 개발함.
- 자바는 포인터 개념이 없어서 외부에서 침입이 불가능한 애플리케이션을 만들 수 있도록 해준다.
- JVM 덕분에 자바의 버전만 동일하다면, 동일한 프로그램은 어떤 플랫폼에서도 실행할 수 있다.
- JVM / JDK / JRE / JIT 막연하게 알던 내용을 정리하였다.

```
JDK — Java Development Kit  (개발 + 실행)
│
├── 개발 도구
│     javac, jar, javadoc, javap, jdb, jshell, jlink
│
└── JRE — Java Runtime Environment  (실행만 가능)
      │
      ├── JVM
      │     클래스 로더 / 실행 엔진 · GC / 메모리 영역
      │
      └── 표준 라이브러리
            java.lang, java.util, java.io, java.net

```

| 구분 | JDK | JRE | JVM | JIT |
|---|---|---|---|---|
| 계층 | 패키지 (최상위) | 패키지 | 실행 엔진 본체 | **JVM 내부 부품** |
| 정체 | 개발자용 설치 패키지 | 실행자용 설치 패키지 | 바이트코드 실행 가상 머신 | 런타임 컴파일러 |
| 핵심 역할 | 컴파일 + 실행 + 개발 도구 | 실행 환경 제공 | 로딩 · 검증 · 실행 · GC | 핫스팟을 네이티브로 변환 |
| 언제 동작 | 빌드 시점 | 실행 시점 | 실행 시점 (전 구간) | 실행 중, 임계치 도달 후 |
| 입력 → 출력 | `.java` → `.class` | `.class` → 실행 | `.class` → 실행 | 바이트코드 → 네이티브 |
| 사용자가 인지? | 설치하고 명령어로 씀 | 설치만 함 | 프로세스로 존재 | 보이지 않음 (자동) |
