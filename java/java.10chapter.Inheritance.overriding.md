## 배운 것
자바의 상속과 overriding

## 왜 중요한가?
자바는 매개 변수 있는 생성자를 만들었을 경우에는 기본 생성자를 자동으로 만들어지지 않음으로 인한 오류가 발생. super()함수 호출하여 부모 클래스 생성자의 매개변수에 맞는 타입 지정하여 오류 방지 가능 또는 컴파일 시 자동 생성

overriding된 메소드의 접근 제어자는 부모 클래스에 있는 메소드와 달라도 되지만, 접근 권한이 확장되는 경우에만 허용된다. 접근 권한이 축소될 경우에는 컴파일 에러가 발생한다.

## 예시 (패키지 선언)

C++과 Java 상속 비교

| 개념 | Java | C++ |
|---|---|---|
| 메서드 오버라이딩 | 기본적으로 다 됨 | virtual 키워드 필요 |
| 부모 생성자 호출 | super() 자동 호출 | 자동 호출 (명시 가능) |
| 다중 상속 | ❌ 불가 (인터페이스는 가능) | ✅ 가능 |


C++과 Java overriding 비교

| | C++ | Java |
|---|---|---|
| 접근 범위 확장 (protected → public) | ✅ | ✅ |
| 접근 범위 축소 (public → protected) | ✅ | ❌ |
