# TIL - JAVA 책: Java의 신

- **날짜:** 2026-05-06
- **분야:** `개발` 

## 배운 것
자바의 패키지와 API 차이

## 왜 중요한가?
패키지와 API 정확한 개념을 알아야 글을 읽을 때나 소통할 때 오해가 없다.

## 새롭게 알게 된 정보
패키지: 클래스들을 담는 폴더 (구조)
API: 그 안의 클래스/메서드들이 제공하는 사용 약속 (인터페이스)

## 예시 

```
📚 도서관 (Java 표준 라이브러리)
├── 📁 소설 책장 (java.lang)        ← 패키지
│   ├── 📖 String 책               ← 클래스
│   ├── 📖 Math 책
│   └── 📖 System 책
├── 📁 자료구조 책장 (java.util)    ← 패키지
│   ├── 📖 ArrayList 책            ← 클래스
│   └── 📖 HashMap 책
```

패키지 = "책장" (분류 구조)
클래스 = "책" (실제 내용물)
API = "이 책들을 어떻게 펼쳐서 어떤 페이지를 보면 되는지에 대한 약속"

```
import java.util.ArrayList;
//     └─패키지─┘ └─클래스─┘

ArrayList<String> list = new ArrayList<>();
list.add("hello");        // ← 이 add() 메서드의 사용법이 "API"
list.size();              // ← 이것도 API
```

패키지는 클래스들을 담은 폴더
API는 클래스에 담긴 인터페이스들을 사용하는 방법에 대한 약속
